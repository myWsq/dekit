Implementation Approach

Two approaches for the device simulator — pick one

A

CSS Transform Scale (Recommended)

How: Set iframe to device's exact pixel dimensions (e.g. 390×844), then use transform: scale() to fit it into the available canvas space.

Pros:

  • Pixel-perfect — iframe sees real device dimensions, CSS media queries fire correctly
  • Simple — no need to calculate DPR or viewport meta injection
  • Touch cursor simulation is natural — just change CSS cursor
  • Zoom level is just the scale factor, easy to show/adjust

Cons:

  • Scaled content may appear slightly blurry at non-integer scale factors

This is how Chrome DevTools does it. The iframe is set to device width/height, then scaled down to fit.

B

CSS Zoom + Viewport Meta Injection

How: Resize iframe container to fit available space, inject <meta name="viewport"> into the iframe to trick the page into thinking it's on a smaller screen, use CSS zoom for display scaling.

Pros:

  • Content stays sharp at any size (no transform blur)

Cons:

  • More complex — need to inject viewport meta into iframe pages
  • CSS zoom behavior varies across browsers
  • Requires modifying the Vite plugin to inject viewport tags dynamically
  • Media queries may not fire as expected
